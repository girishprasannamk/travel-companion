# Requirements

The application should:

- Work offline (PWA + localStorage persistence)
- Install on iPhone (manifest, standalone display)
- Support multiple trips (data-driven Trip Engine, no hardcoded trips)
- Store tickets (M2)
- Store documents (M3, offline-accessible)
- Track expenses with INR conversion + analytics (M2)
- Manage itinerary / timeline (M2)
- Capture journal memories (M2)
- Support future AI features (M4)
- Deploy to GitHub Pages, and run unchanged on Vercel / Firebase Hosting
